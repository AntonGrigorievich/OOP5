# ЛР5: Пояснение (однофайловая версия)

Вся лабораторная находится в одном файле: `lab5/main.cpp`.
Это сделано специально, чтобы на защите не переключаться между файлами.

## Структура `main.cpp`

Файл разделён на логические секции (по комментариям):
- `Сущности` — классы `Base` и `Desc`.
- `Демо: передача параметров` — namespace `ParamDemo`.
- `Демо: возврат из функций` — namespace `ReturnDemo`.
- `Демо: умные указатели` — namespace `SmartPtrDemo`.
- `Сценарии демонстрации` — функции `demo*`, которые запускают примеры по блокам.
- `main()` — порядок вызова всех демонстраций.

## Что происходит в каждой секции

### 1) Сущности (`Base`, `Desc`)
- У `Base` есть статическое поле `next_id_`, которое выдаёт уникальный `id` каждому объекту.
- Во всех конструкторах/деструкторах печатается лог, чтобы видно было жизненный цикл.
- Есть пары методов для сравнения:
  - невиртуальные (`nonVirtualMethod`, `method2NonVirtual`),
  - виртуальные (`virtualMethod`, `method2Virtual`).
- Методы `method1CallsMethod2NonVirtual` и `method1CallsMethod2Virtual` показывают, как отличается вызов невиртуального и виртуального метода внутри базового класса.
- `classname()` и `isA()` нужны для ручной проверки типа.

`Desc` наследуется от `Base` и:
- переопределяет виртуальные методы,
- добавляет `descOnly()`,
- расширяет `isA()` так, чтобы `Desc` считался и `Desc`, и `Base`.

### 2) `ParamDemo`
Реализованы три функции из методички:
- `func1(Base obj)` — передача по значению (копирование/срезка).
- `func2(Base* obj)` — передача указателя (сохранение полиморфизма).
- `func3(Base& obj)` — передача ссылки (сохранение полиморфизма).

Внутри показывается:
- работа `virtualMethod`,
- разница `isA + static_cast` и `dynamic_cast`.

### 3) `ReturnDemo`
Реализованы шесть функций возврата:
- `func1/func2/func3` работают со `static` локальным объектом,
- `func4/func5/func6` работают с динамическим объектом `new`.

Секция нужна, чтобы увидеть:
- когда происходит копирование при возврате,
- где появляется ответственность за `delete`,
- какие варианты потенциально опасны.

### 4) `SmartPtrDemo`
Показывает `unique_ptr` и `shared_ptr`:
- `takeUnique` принимает владение,
- `takeShared` читает объект без захвата владения,
- `makeUniqueBase` и `makeSharedBase` возвращают умные указатели из функций.

По выводу видно:
- у `unique_ptr` владелец ровно один,
- у `shared_ptr` есть счётчик `use_count`.

### 5) Сценарии `demo*`
Каждая `demo`-функция запускает один логический блок:
- `demoMethodHidingAndVirtual` — сравнение virtual/non-virtual.
- `demoRequiredConstructorsAndVirtualDtor` — требуемые конструкторы + удаление `Desc` через `Base*`.
- `demoClassnameAndIsAAndCasts` — проверки типа и приведения.
- `demoParamsLifecycle` — передача параметров.
- `demoReturnLifecycle` — возврат из функций.
- `demoSmartPointers` — умные указатели.

### 6) `main()`
`main()` просто вызывает demo-блоки в удобном для защиты порядке.

## Как запускать

```bash
cd lab5
g++ -std=c++17 -Wall -Wextra -pedantic main.cpp -o lab5
./lab5
```
